What Sudo Dodo IS.The one-stop Linux shop. The DistroWatch killer. The place on the internet where anyone — Windows refugee or 20-year Arch veteran — goes for any and everything Linux. Design rules — all agents follow all of these, always

Rule 1 — Monospace everything.Font: 'Courier New', 'Lucida Console', monospace. Body text, navigation, meta, post titles — everything. No web font imports. No sans-serif anywhere in the UI.
Rule 2 — The orange header bar is sacred.Every page: background #ff6600, text #000000, bold monospace. Logo left, nav middle, user actions right. Exactly like Hacker News. Never taller, fancier, or animated.
Rule 3 — Text is the UI.Buttons: [ action ] style. Links underlined on hover. No rounded buttons. No filled buttons. No icon-only buttons. The label IS the button.
Rule 4 — Borders not shadows.Every container uses 1px solid border only. No box-shadow ever. Nothing floats. Nothing has depth. Colour from active theme variable.
Rule 5 — Information density over whitespace.Model: old.reddit.com compact mode + Hacker News. Rows are tight. Padding minimal. Maximum content visible without scrolling. No 48px padding, no centered content with 40% margins.
Rule 6 — Theme system is CSS variables only.All colours defined as CSS variables on :root. Each theme class overrides those variables only. Change body class → everything updates. Required variables: --bg, --bg2, --fg, --fg2, --fg3, --accent, --link, --border, --score-pos, --score-neg.
Rule 7 — Three columns desktop, one column mobile.Left sidebar ~180px, centre feed flexible, right sidebar ~200px. Below 640px: both sidebars hidden, single column. Plain CSS flexbox or grid. No framework.
Rule 8 — Official logos only, never decorative.Distro logos on distro pages and rankings. Official images only. No AI art, no stock photos. Where no logo exists, use text abbreviation in a box.
Rule 9 — Six themes are required.Default (dark, orange), Matrix (black, green), Amber (dark, amber), Phosphor (dark, bright green), C64 (blue, white), Paper (light, sepia). Every page works in all six. Test in all six before marking any job done.
Rule 10 — 🦤 is the only mascot.Used as text emoji. In the logo. On 404. On the wizard. In the footer. Never as a large image. Never animated. Never in a fancy frame. Just 🦤 in context.

Build rules — the law that governs every job

Rule 1 — One job. One file set. One test.Every prompt does exactly one thing. Names specific files. Ends with a test. If the test fails, the job is not done. Never give an agent two jobs in one prompt.
Rule 2 — No UI without a working backend.No button, link, or interactive element not connected to a real working system. No placeholder handlers. No hardcoded data. If the data doesn't exist, the UI doesn't exist yet.
Rule 3 — Prove it with a database row, not a screenshot.Auth done? Show the Supabase user row. Vote done? Show the votes table row. UI done? Vote, reload, show the count changed in the DB.
Rule 4 — Design rules apply to every frontend job.Every frontend job must pass the anti-patterns checklist. Any frontend job that fails is rejected and restarted.
Rule 5 — Test in all six themes before marking a UI job done.Switch through all six themes. If anything looks broken in any theme, the job is not done.
