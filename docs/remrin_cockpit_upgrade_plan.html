Rem's cockpit audit — current state vs Claude

Working well: The sidebar with chat history, New Chat button, and Cockpit Settings is structurally correct — same fundamental layout as Claude. The "Neural Link" status indicator is a nice branded touch. The persistent memory positioning is excellent.
Critical gap — No artifact/canvas system. This is the single biggest difference between the current cockpit and Claude. Claude renders code, tables, charts, and rich content in a separate right-side panel ("artifact") rather than inline. This is what makes Claude feel like a professional tool rather than a chat window. Without this, formatted output like pricing tables or code just dumps into the chat as raw text.
Critical gap — No markdown rendering. If the AI outputs **bold**, headers, bullet lists, or code blocks, they need to render as formatted HTML, not raw asterisks and backticks. This is what makes Claude's responses feel rich and readable vs a plain text dump.
Missing — Inline code highlighting. Claude uses syntax-highlighted code blocks with a copy button and language label. This is a high-visibility feature for a tool positioning itself as a Claude alternative.
Missing — Streaming text rendering. Claude types out responses token by token with a cursor. If the cockpit shows nothing then dumps the full response, it feels slow and broken even if the latency is identical.
Missing — Message actions. Claude shows Copy, Thumbs up/down, Retry, and Edit on hover over each message. These small interactions signal quality and build trust.
Opportunity — Model switcher. The cockpit shows "DeepSeek V3" — this should be a visible, clickable model selector exactly like Claude's model dropdown. Let users choose between available models. This is a power-user feature that reinforces the "professional tool" positioning.
Opportunity — Typography. Claude uses a clean sans-serif for UI chrome and a slightly different weight for AI responses. The contrast between UI text and message text makes responses feel more distinct and readable. Worth matching deliberately.
Nice to have — Thinking indicator. The animated "Rem is thinking..." with a pulsing indicator between user message and AI response reduces perceived latency significantly.

Feature implementation roadmap — priority order

📄
Markdown rendering P1 — Do first
Install react-markdown + remark-gfm. Renders **bold**, # headers, bullet lists, tables, and blockquotes as styled HTML. Without this nothing else matters — raw asterisks break trust immediately.
⚡
Streaming response (token by token) P1 — Do first
Use the AI SDK's streaming endpoint. Render each token as it arrives. Add a blinking cursor ▋ at the end of the in-progress message. Transforms perceived speed dramatically.
💻
Syntax-highlighted code blocks P1 — Do first
Use react-syntax-highlighter or shiki. Every code block gets: language label top-left, copy button top-right, dark background, coloured syntax. One of the most visible quality signals.
🖼️
Artifact / canvas panel P2 — High value
Side panel that opens when AI returns rich content (HTML, React components, long structured documents). Keeps chat clean. Exactly how Claude handles artifacts. Triggered automatically by content type or by user request. Has its own toolbar: copy, download, preview/code toggle.
🖱️
Message hover actions P2 — High value
On hover over any AI message: Copy full message, Thumbs up, Thumbs down, Retry (regenerate), Edit (for user messages). Appears as a floating toolbar above the message. Signals quality, enables feedback loop.
🤖
Model selector dropdown P2 — High value
Clickable model name in the header or input area. Dropdown shows available models with labels (Fast / Smart / Most Capable). Selected model persists per conversation. Pro-only models show a lock icon for free users.
💬
Thinking / typing indicator P3 — Polish
Animated pulsing dots or "Rem is thinking..." text shown between user message and AI response while the stream hasn't started yet. Eliminates the dead zone where nothing is happening.
📎
File / image upload in chat P3 — Polish
Drag-and-drop or paperclip button in the input area. Supports images (send to vision model), PDFs (extract text), and text files. Preview thumbnail shown above the input before sending. Tier-gated for large files.
🔍
Chat search P4 — Nice to have
Search across all previous conversations. Highlights matching messages. Essential once a user has months of history.
📤
Export conversation P4 — Nice to have
Export current chat as Markdown, PDF, or plain text. Tier-gated (Soul Weaver+). Adds perceived value to paid tiers.
